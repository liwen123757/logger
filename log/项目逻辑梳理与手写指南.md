# bitlog 项目逻辑梳理与手写指南

> 目标：先理解项目模块关系，再按“可运行最小集”一步步手写。

## 1. 项目在做什么

这是一个 **C++ 日志库**，核心能力：

- 支持日志级别（DEBUG/INFO/WARN/ERROR/FATAL）
- 支持格式化输出（时间、线程、文件行号、正文等）
- 支持多种落地方式（控制台、文件、滚动文件）
- 支持同步/异步两种日志器
- 通过 Builder 模式配置日志器，并由 loggerManager 统一管理

---

## 2. 源码目录与职责

- `log/logs/level.hpp`：定义日志级别与级别字符串转换。
- `log/logs/message.hpp`：定义日志消息对象（名称、级别、正文、位置信息等）。
- `log/logs/formatter.hpp`：解析格式串并格式化 `LogMsg`。
- `log/logs/sink.hpp`：定义不同落地器（stdout/file/roll）。
- `log/logs/queue.hpp` + `log/logs/buffer.hpp`：异步队列与缓冲区。
- `log/logs/looper.hpp`：异步线程循环（前台 push，后台消费）。
- `log/logs/logger.hpp`：日志器核心（同步/异步、Builder、Manager）。
- `log/logs/bitlog.h`：对外 API + 快捷宏。
- `log/example/logger.cc`：完整使用示例（已做逐行注释）。

---

## 3. 一条日志的执行路径（最重要）

以 `LOG_ERROR(lp, "%s", "hello")` 为例：

1. 宏展开成 `lp->error(__FILE__, __LINE__, fmt, ...)`
2. `Logger::error` 先检查级别是否需要打印
3. `Logger::log` 用 `vasprintf` 生成正文字符串
4. 组装 `LogMsg`
5. `Formatter::format` 按 pattern 拼装最终日志文本
6. 调 `logIt`：
   - 同步：当前线程直接写各个 sink
   - 异步：丢进 `AsyncLooper` 队列，后台线程批量写 sink

---

## 4. 手写顺序（推荐）

如果你要“从零手敲”这个项目，建议顺序：

1. `level.hpp`（最简单）
2. `message.hpp`（日志结构体）
3. `sink.hpp`（先做 stdout，再做 file）
4. `formatter.hpp`（先做 `%m%n`，再扩展）
5. `logger.hpp` 里的同步日志器
6. `loggerManager` + Builder
7. 异步队列/looper（最后做）
8. `bitlog.h` 宏封装
9. 写 example 验证

---

## 5. 你接下来可以这样练

- 第 1 轮：只做 **同步 + stdout + 固定格式**，确保跑通。
- 第 2 轮：加 **file sink + 可配置格式串**。
- 第 3 轮：加 **异步日志器 + 滚动文件**。
- 第 4 轮：补异常场景（目录不存在、文件打开失败、格式错误）。

---

## 6. 关于“每一行都注释”的建议

工程里“每行注释”会降低可读性，行业里通常做法是：

- 关键类/函数写“职责注释”
- 复杂分支写“为什么这么做”
- 公共 API 写“输入/输出/线程安全性”

不过为了教学，你可以先在示例文件做逐行注释（本仓库的 `log/example/logger.cc` 已按这种方式处理），等理解后再精简为高价值注释。
